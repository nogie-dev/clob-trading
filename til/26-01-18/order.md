 # 설명

  ## 문제 정의
  - 주문 번호
    - RequestOrder 해시로 생성하려 했으나 변동 가능성 있는 필드(가격/수량)는 제외.
    - 동일 포지션·가격·수량에서 해시를 사용하면 결정론적인 값이어서 이더리움의 EOA nonce를 차용

  ## 인덱싱
  - 현재 주문은 이중 연결 리스트 큐에 쌓여 있음.
  - 주문번호(key) → 리스트 노드 포인터(value)로 매핑해 즉시 접근/삭제.
    - 모든 주문을 인덱싱하면 주문 수에 비례해 메모리가 늘어남.

  ## 주문 수정 시 큐 우선순위/탐색 처리
  - 가격 수정: 현재 큐에서 제거하고 새 호가 큐에 재삽입.
  - 수량 증가(후순위): 기존 주문을 취소 후 새로 넣거나, 기존 주문은 그대로 두고 증가분만 별도 주문으로 추가해 뒤쪽
  에 배치.
  - 수량 감소(순위 유지): 해당 주문을 찾아 수량만 줄이고 레벨 누적도 함께 감소.
